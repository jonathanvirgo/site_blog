# UI/UX: Content Crawler (Admin)

> **Standard**: This document follows [_wireframe_standard.md](./_wireframe_standard.md)

---

## 1. Overview

The Content Crawler admin interface allows administrators to import products and articles from external websites. The interface provides URL input, preview of extracted content, and approval workflow.

### User Journey
```
Configure Source â†’ Crawl Category Pages â†’ Extract Detail Data â†’ Preview â†’ Approve â†’ Save
```

---

## 2. User Stories

- As an **admin**, I want to paste a URL and extract product/article data automatically
- As an **admin**, I want to preview and edit extracted content before saving
- As an **admin**, I want to see the status of all crawl jobs in a queue
- As an **admin**, I want to configure CSS selectors for different source websites
- As an **admin**, I want to map category URLs to specific categories with status
- As an **admin**, I want to test selectors before saving configuration

## 7. API Endpoints

### Test APIs
- `POST /api/crawler/test-list` - Test category page selectors
- `POST /api/crawler/test-selector` - Test individual selector

### Source APIs
- `GET /api/crawler/sources` - List sources
- `POST /api/crawler/sources` - Create source
- `GET /api/crawler/sources/[id]` - Get source
- `PUT /api/crawler/sources/[id]` - Update source
- `DELETE /api/crawler/sources/[id]` - Delete source

## 8. States & Feedback

### Loading States
- URL submission: Button shows spinner
- Test selector: Inline loading
- Save: Button disabled with spinner

### Success Feedback
- Test passed: Green checkmark with result
- Save successful: Toast notification

### Error States
- Invalid URL: Inline error
- Selector not found: Red warning
- Save failed: Toast with retry

## 9. Mobile Responsiveness

- **Tabs**: Horizontal scroll on mobile
- **Mapping cards**: Full width stack
- **Selector inputs**: Full width

- Form labels for all inputs
- Error messages linked to inputs
- Keyboard navigation for tabs
