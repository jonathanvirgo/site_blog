# UI/UX: Media Library (Image Management)

> **Standard**: This document follows [../_wireframe_standard.md](../_wireframe_standard.md)

---

## 1. Overview

Media Library provides centralized management for all uploaded images with:
- Drag & drop upload
- Automatic responsive variants via Cloudinary
- Cropping with aspect ratio presets
- Folder organization

### Cloudinary Integration
| Feature | Transformation |
|---------|---------------|
| Auto format | `f_auto` (WebP/AVIF) |
| Auto quality | `q_auto` (optimal compression) |
| AI cropping | `c_auto,g_auto` (smart focus) |
| Resize | `w_{width}` (on-the-fly) |

---

## 2. User Stories

| ID | As a... | I want to... | So that... |
|----|---------|--------------|------------|
| US-01 | Editor | Upload images via drag & drop | I can quickly add media |
| US-02 | Admin | Crop images to different ratios | They fit various layouts |
| US-03 | Developer | Get responsive image URLs | I use srcset properly |
| US-04 | Admin | Organize images in folders | Media is easy to find |

## 4. Components

### 4.1 Image Upload
```typescript
interface ImageUploadProps {
  onUpload: (files: File[]) => Promise<UploadResult[]>;
  maxSize: number; // bytes
  acceptedTypes: string[];
  multiple?: boolean;
  folder?: string;
}

interface UploadResult {
  id: string;
  publicId: string;
  url: string;
  width: number;
  height: number;
  size: number;
  variants: ResponsiveVariant[];
}
```

### 4.2 Image Cropper
```typescript
interface ImageCropperProps {
  image: string;
  aspectRatio?: number;
  onCrop: (croppedImage: CropResult) => void;
  presets: AspectRatioPreset[];
}

type AspectRatioPreset = {
  name: string;
  ratio: number; // width / height
  icon: string;
};
```

### 4.3 Responsive Variant
```typescript
interface ResponsiveVariant {
  name: 'xs' | 'sm' | 'md' | 'lg' | 'xl' | '2xl';
  width: number;
  url: string;
  size: number; // bytes
}

const VARIANT_WIDTHS = {
  xs: 320,
  sm: 480,
  md: 768,
  lg: 1024,
  xl: 1280,
  '2xl': 1920
};
```

## 5. Data Requirements

### API Endpoints
| Endpoint | Method | Purpose |
|----------|--------|---------|
| `POST /admin/media/upload` | POST | Upload images |
| `GET /admin/media` | GET | List all media |
| `GET /admin/media/:id` | GET | Get media details |
| `PATCH /admin/media/:id` | PATCH | Update alt text, folder |
| `DELETE /admin/media/:id` | DELETE | Delete media |
| `POST /admin/media/:id/crop` | POST | Create cropped version |
| `GET /admin/media/folders` | GET | List folders |
| `POST /admin/media/folders` | POST | Create folder |

### Database Schema
```prisma
model MediaAsset {
  id           String   @id @default(uuid())
  publicId     String   @unique
  filename     String
  url          String
  width        Int
  height       Int
  size         Int
  mimeType     String
  alt          String?
  folderId     String?
  folder       MediaFolder? @relation(fields: [folderId], references: [id])
  variants     Json     // { xs: url, sm: url, ... }
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt
}

model MediaFolder {
  id       String       @id @default(uuid())
  name     String
  parentId String?
  parent   MediaFolder? @relation("FolderTree", fields: [parentId], references: [id])
  children MediaFolder[] @relation("FolderTree")
  assets   MediaAsset[]
}
```
